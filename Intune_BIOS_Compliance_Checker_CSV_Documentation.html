BIOS Compliance Checker CSV-Modus v1.1

Eine login-freie, vollständig CSV-basierte Variante des Intune BIOS Compliance Checkers. Statt Daten live via Microsoft Graph API abzurufen, werden zwei CSV-Exporte aus dem Intune Admin Center importiert. Kein Azure AD, kein App-Registration, kein Token — der Browser verarbeitet alles lokal.

Was ist das Tool?

Das Tool prüft Windows-Geräte aus Intune auf BIOS-Compliance bezüglich des Windows Secure Boot 2023 Zertifikats-Updates (Ablauf Juni 2026). Es vergleicht die installierte BIOS-Version mit den von Dell und HP veröffentlichten Mindestversionen.

Im Gegensatz zur Graph-basierten Variante benötigt dieses Tool keine Anmeldung und keine Azure AD App-Registrierung. Die Gerätedaten werden stattdessen aus einem CSV-Export aus dem Intune Admin Center geladen.

Geräte-CSV exportieren

Der CSV-Export erfolgt im Intune Admin Center:

1

Intune Admin Center öffnen

Navigiere zu intune.microsoft.com → Geräte → Alle Geräte.

2

Export starten

Klick auf Exportieren (oben rechts in der Geräteliste). Intune erstellt einen vollständigen Export aller Geräte inkl. Hardwareinventardaten.

3

CSV herunterladen

Der Export wird als ZIP-Datei bereitgestellt. Die enthaltene CSV-Datei direkt ins Tool laden.

⚠️
Hardwareinventar muss aktiviert sein

Die Spalte SystemManagementBIOSVersion ist nur befüllt, wenn das Hardwareinventar für die Geräte in Intune aktiviert ist und die Geräte die Daten bereits gemeldet haben. Geräte ohne BIOS-Wert werden als nicht geprüft angezeigt.

Secure Boot CSV exportieren

Optional kann ein zweiter CSV-Export aus dem Intune Reporting importiert werden, der Secure Boot Status und Zertifikat-Status enthält:

Intune Admin Center → Reports → Windows Autopatch → Windows quality updates → Reports → Secure Boot status → Exportieren

Das Tool verknüpft die Daten automatisch über den Gerätenamen (case-insensitive). Die Secure Boot- und Zertifikat-Spalten bleiben leer (—), solange kein CSV importiert wurde.

CSV-Import

Das Tool bietet zwei Drop-Zonen nebeneinander:

  • Geräte-CSV (Pflicht) — per Drag & Drop oder Klick
  • Secure Boot Status CSV (optional) — per Drag & Drop oder Klick

Beide CSVs können in beliebiger Reihenfolge importiert werden. Wird die Secure Boot CSV nach dem Starten der Analyse nachgeladen, aktualisiert sich die Tabelle automatisch.

ℹ️

Das Tool filtert beim Parsen automatisch: Nicht-Windows-Geräte (Spalte OS ≠ Windows) und MDE-only Geräte (Spalte Managed by = MDE) werden ausgeschlossen.

Analyse starten

Nach dem Import der Geräte-CSV wird die Schaltfläche Analyse starten aktiv. Ein Klick darauf:

  1. Befüllt die Dropdown-Filter mit den vorhandenen Herstellern und Modellen
  2. Führt den BIOS-Compliance-Abgleich für alle Geräte durch
  3. Zeigt die Ergebnistabelle und Summary-Kacheln an
  4. Scrollt automatisch zu den Ergebnissen

Mit Zurücksetzen werden alle importierten Daten gelöscht und das Tool in den Ausgangszustand versetzt.

CSV-Parsing-Logik

Geräte-CSV

Das Tool verwendet einen vollständigen RFC 4180-kompatiblen CSV-Parser, der mit Anführungszeichen, eingebetteten Kommas und Zeilenumbrüchen in Feldern umgehen kann. Beim Parsen werden automatisch gefiltert:

  • Nicht-Windows-Geräte — Spalte OS muss Windows enthalten (case-insensitive)
  • MDE-only Geräte — Spalte Managed by = MDE werden ausgeschlossen

Die Spalte SystemManagementBIOSVersion wird über eine Suche nach dem Schlüsselwort bios im Header erkannt — unabhängig von der genauen Gross-/Kleinschreibung.

Secure Boot CSV

Die Secure Boot CSV wird ebenfalls RFC 4180-konform geparst. Die Verknüpfung mit den Gerätedaten erfolgt über den Gerätenamen (Spalte Device name) als Join-Key, case-insensitiv. Erwartete Spalten:

  • Device name — Join-Key
  • Secure Boot enabled — Yes / No / Unknown
  • Certificate status — Up to date / Not up to date / Not applicable

BIOS-Compliance-Prüfung

Versionsvergleich

Die aktuelle BIOS-Version wird segmentweise numerisch mit der Mindestversion verglichen — nicht alphabetisch:

compareVersions("01.10.00", "01.09.00")  →  1  (aktuell >= minimum ✓)
compareVersions("1.8.0",    "1.10.0")    → -1  (veraltet ✗)
compareVersions("1.10.0",   "1.10.0")   →  0  (exakt gleich ✓)

Modellabgleich (Normalisierung + Fuzzy)

Intune liefert Modellnamen oft in ausführlicher Form. Der Abgleich läuft in drei Stufen:

StufeMethodeBeispiel
1 Normalisierung Störwörter entfernen (Notebook PC, Inch, Laptop PC, …) HP EliteBook 860 16 inch G10 Notebook PC → hp elitebook 860 16 g10
2 Substring Normalisierter DB-Schlüssel ist in Intune-Namen enthalten (oder umgekehrt) hp elitebook 840 g8 ⊂ hp elitebook 840 g8 sonderconfig
3 Fuzzy (Jaccard) Wortmengen-Ähnlichkeit ≥ 60 % hp elitebook 840-g8 ↔ hp elitebook 840 g8

Ein Tooltip auf der Mindestversions-Zelle zeigt an, welcher DB-Eintrag für den Match verwendet wurde.

BIOS-Versionsextraktion

HP gibt BIOS-Versionen mit der BIOS Family als Präfix zurück. Das Tool extrahiert die numerische Version automatisch:

"V70 ver. 01.10.00"  →  "01.10.00"
"T76 Ver. 01.22.00"  →  "01.22.00"
"01.10.00"           →  "01.10.00"   (unverändert)
"1.32.1"             →  "1.32.1"     (unverändert)

Der ursprüngliche Rohwert wird im CSV-Export als separate Spalte ausgegeben und als Tooltip auf der BIOS-Zelle angezeigt.
