BIOS Compliance Checker CSV Mode v1.1

A login-free, fully CSV-based variant of the Intune BIOS Compliance Checker. Instead of fetching data live via the Microsoft Graph API, two CSV exports from the Intune Admin Center are imported. No Azure AD, no app registration, no token — the browser processes everything locally.

What is this tool?

The tool checks Windows devices from Intune for BIOS compliance with respect to the Windows Secure Boot 2023 certificate update (expiring June 2026). It compares the installed BIOS version against the minimum versions published by Dell and HP.

Unlike the Graph-based variant, this tool requires no sign-in and no Azure AD app registration. Device data is instead loaded from a CSV export from the Intune Admin Center.

Export the device CSV

The CSV export is done in the Intune Admin Center:

1

Open Intune Admin Center

Navigate to intune.microsoft.com → Devices → All devices.

2

Start the export

Click Export (top right of the device list). Intune generates a full export of all devices including hardware inventory data.

3

Download the CSV

The export is provided as a ZIP file. Extract the CSV file and load it directly into the tool.

⚠️
Hardware inventory must be enabled

The SystemManagementBIOSVersion column is only populated if hardware inventory is enabled for devices in Intune and the devices have already reported the data. Devices without a BIOS value are shown as not checked.

Export the Secure Boot CSV

Optionally, a second CSV export from Intune Reporting can be imported, containing Secure Boot status and certificate status:

Intune Admin Center → Reports → Windows Autopatch → Windows quality updates → Reports → Secure Boot status → Export

The tool automatically joins the data by device name (case-insensitive). The Secure Boot and certificate columns remain empty (—) until a CSV is imported.

CSV import

The tool provides two drop zones side by side:

  • Device CSV (required) — drag & drop or click to browse
  • Secure Boot Status CSV (optional) — drag & drop or click to browse

Both CSVs can be imported in any order. If the Secure Boot CSV is loaded after the analysis has already started, the table updates automatically.

ℹ️

The tool automatically filters during parsing: non-Windows devices (column OS ≠ Windows) and MDE-only devices (column Managed by = MDE) are excluded.

Run analysis

Once the device CSV has been imported, the Run analysis button becomes active. Clicking it:

  1. Populates the dropdown filters with available manufacturers and models
  2. Runs the BIOS compliance check for all devices
  3. Displays the results table and summary tiles
  4. Automatically scrolls to the results

Use Reset to clear all imported data and return the tool to its initial state.

CSV parsing logic

Device CSV

The tool uses a fully RFC 4180-compliant CSV parser that handles quoted fields, embedded commas and line breaks within fields. The following are automatically filtered during parsing:

  • Non-Windows devices — column OS must contain Windows (case-insensitive)
  • MDE-only devices — column Managed by = MDE are excluded

The SystemManagementBIOSVersion column is identified by searching for the keyword bios in the header — regardless of exact capitalisation.

Secure Boot CSV

The Secure Boot CSV is also parsed RFC 4180-compliant. The join with device data is performed using the device name (column Device name) as the join key, case-insensitively. Expected columns:

  • Device name — join key
  • Secure Boot enabled — Yes / No / Unknown
  • Certificate status — Up to date / Not up to date / Not applicable

BIOS compliance check

Version comparison

The installed BIOS version is compared against the minimum version using numeric segment-by-segment comparison — not alphabetically:

compareVersions("01.10.00", "01.09.00")  →  1  (current >= minimum ✓)
compareVersions("1.8.0",    "1.10.0")    → -1  (outdated ✗)
compareVersions("1.10.0",   "1.10.0")   →  0  (exact match ✓)

Model matching (normalisation + fuzzy)

Intune often returns verbose model names. The matching runs in three stages:

StageMethodExample
1 Normalisation Strip noise words (Notebook PC, Inch, Laptop PC, …) HP EliteBook 860 16 inch G10 Notebook PC → hp elitebook 860 16 g10
2 Substring Normalised DB key is contained in the Intune name (or vice versa) hp elitebook 840 g8 ⊂ hp elitebook 840 g8 special
3 Fuzzy (Jaccard) Word-set similarity ≥ 60 % hp elitebook 840-g8 ↔ hp elitebook 840 g8

A tooltip on the minimum version cell shows which DB entry was used for the match.

BIOS version extraction

HP returns BIOS versions with the BIOS family as a prefix. The tool extracts the numeric version automatically:

"V70 ver. 01.10.00"  →  "01.10.00"
"T76 Ver. 01.22.00"  →  "01.22.00"
"01.10.00"           →  "01.10.00"   (unchanged)
"1.32.1"             →  "1.32.1"     (unchanged)

The original raw value is exported as a separate column in the CSV and shown as a tooltip on the BIOS cell.
